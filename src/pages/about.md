我是高瑞林，一个写了快十年代码的开发者。2025 年开始把 Claude Code 正式引入日常开发流程，到现在已经用了一年多。

这个站记录的是我从最初的"哇这东西好厉害"，到中间反复被坑——AI 改了不该改的文件、生成了根本不存在的 API、一次性产出 500 行不敢合并的代码——再到后来逐渐摸清楚怎么跟它协作的完整过程。

## 为什么做这个站

我看过不少关于 Claude Code 的内容，大致分两类：一类是官方文档的中文翻译，该有的信息都有但缺少实际场景；另一类是社交媒体上的效率展示帖，"五分钟搞定一个项目"之类的，看完热血沸腾但回去一用发现不是那么回事。

真正在项目里长期使用这个工具之后，我发现最难的不是学命令——命令翻翻文档就会了——而是搞清楚几个核心问题：

- 一个需求下来，哪些部分适合交给 AI，哪些必须自己把控？
- 上下文窗口就那么大，怎么安排信息才能让 AI 的输出质量不掉？
- AI 写的代码看着对，实际跑起来有各种边界问题，怎么审查？
- 项目越做越复杂，前面的对话要不要保留，什么时候该开新会话？

这些问题没有标准答案，但可以通过实践总结出一套可复用的方法。这个站就是在做这件事。

## 这里有什么

目前整理了六大板块：

**入门篇**从安装配置开始，讲 CLI 用法、prompt 写法、文件操作、MCP 入门、费用攻略。就算你没用过任何 AI 编程工具也能跟上。

**思维篇**是我花精力最多的部分。任务拆解、上下文管理、代码审查、错误恢复、架构决策——这些不是 Claude Code 专属的技巧，而是软件工程的基本功，在 AI 协作场景下更加关键。

**实战篇**是完整的项目开发复盘。从零搭建项目、重构巨型组件、排查线上 Bug，每篇都带完整的踩坑和修复过程。还包括 CLAUDE.md 模板合集和提示词库。

**工具对比**客观横评 Claude Code、Cursor、Copilot、Windsurf 等 AI 编程工具，帮你做出最适合自己的选择。

**副业指南**分享用 AI 工具开启技术变现的实操方法，从搭建网站到 AdSense 变现，从接私活提效到公众号运营。

**进阶篇**面向有经验的开发者，讲 Headless 模式、管道操作、自动化集成等高级用法。

## 关于内容更新

我不追求更新频率，每篇教程从初稿到发布通常要改三四遍。有些文章写到一半发现自己的方法有问题，就推翻重来。所以更新不会很快，但发出来的内容我尽量保证是经过验证的。

所有内容免费阅读，代码示例可以直接拿去用。
