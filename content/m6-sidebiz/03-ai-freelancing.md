# AI 接单与自动化：用 Claude Code 提升接私活效率

## 背景

我从 2022 年开始在业余时间接私活，主要做中小企业的网站开发和数据相关的小工具。2025 年开始大量使用 Claude Code 之后，接单效率有了质的提升。

这篇文章分享我在接私活过程中使用 AI 的真实经验，包括在哪里找活、怎么报价、怎么用 AI 加速开发、以及几个完整的项目案例。

## 在哪里找活

### 国外平台

**Upwork**

这是我主力平台。优点是项目多、付款有保障、可以积累评价。缺点是竞争激烈、平台抽成高（前 500 美元抽 20%）。

我在 Upwork 上主要接两类活：
- Web 应用开发（React/Next.js）
- 数据处理和自动化脚本

刚开始很难接到活，因为没有评价。我的破冰策略是：前 3 单报价压低 30-40%，快速积累好评。有了 5 星评价之后，单价可以慢慢涨上去。

**Fiverr**

适合做标准化的服务。比如"帮你搭建一个 Next.js 网站"、"帮你写一个 Python 爬虫"。我在 Fiverr 上开了几个固定服务（gig），定价明确，客户下单后直接做。

Fiverr 的好处是不用花时间投标，客户会主动找上来。坏处是客户预期管理比较难，经常有人想用最低价格买最多的服务。

### 国内平台

**程序员客栈**

国内最大的 IT 外包平台之一。项目以国内中小企业为主，沟通方便，但价格比国外平台低不少。

**电鸭社区**

主打远程工作，有一些兼职和项目外包的机会。社区氛围不错，项目质量也还行。

**闲鱼/朋友圈**

别笑，闲鱼真的能接到活。搜"网站开发"、"小程序开发"，能看到不少需求。朋友圈更是，朋友介绍的活通常沟通成本最低、付款最靠谱。

### 我的接单策略

我现在的收入构成大概是：

- Upwork：50%（单价高，但需要投标）
- Fiverr：20%（自动来单，省心）
- 朋友介绍：20%（质量最高）
- 其他：10%

每月花在接单上的总时间大约 30-40 小时，月收入在 8,000-15,000 元之间浮动。

## 报价技巧

### 按项目还是按小时？

我的经验是：**尽量按项目报价**。

按小时计费看起来公平，但有两个问题：
1. 用了 AI 之后你的效率提升了 3 倍，如果按小时计费，收入反而降了
2. 客户会不断质疑你的工时是否合理

按项目报价的好处是，你用 AI 省下来的时间就是你的利润。同样一个项目，以前要 40 小时，现在 15 小时就搞定了，但报价还是按 40 小时的标准报。

### 怎么估算工时

我有一套估算方法：

1. 把项目拆成具体的功能模块
2. 每个模块估算"不用 AI 要多久"
3. 打个 0.35-0.5 的系数（AI 加速后的实际用时）
4. 加上 20% 的沟通和修改缓冲
5. 按"不用 AI"的工时向客户报价

举个例子：

```
项目：企业官网（Next.js + CMS）

功能拆解：
- 首页设计还原            不用AI: 8h  实际: 3h
- 关于我们/联系页面       不用AI: 4h  实际: 1.5h
- 产品展示页（含筛选）    不用AI: 10h 实际: 4h
- 博客系统               不用AI: 8h  实际: 3h
- CMS 集成（Strapi）      不用AI: 6h  实际: 2.5h
- 响应式适配             不用AI: 6h  实际: 2h
- SEO 优化               不用AI: 4h  实际: 1.5h
- 部署和测试             不用AI: 4h  实际: 2h

总计：不用AI 50h / 实际 19.5h
加 20% 缓冲：实际 24h
按 50h x 200 元/h 报价 = 10,000 元
实际投入 24h，时薪约 417 元
```

## 用 Claude Code 加速开发：实战案例

### 案例 1：企业官网项目

**需求**：一个做智能家居的公司要一个产品展示网站，包含首页、产品列表、产品详情、关于我们、联系方式。设计稿已有（Figma）。

**用时**：总计 18 小时（不用 AI 估计要 50 小时）

**具体怎么用 Claude Code 的**：

第一步，初始化项目：

```
用 Next.js 14 App Router 初始化一个企业官网项目。
需要：Tailwind CSS、TypeScript、ESLint。
项目结构要清晰，按功能模块组织。
```

3 分钟搞定项目骨架。

第二步，把设计稿还原成组件。这是最省时间的一步。我把 Figma 设计稿截图给 Claude Code，让它生成对应的 React 组件。

```
这是首页的设计稿截图。请帮我用 React + Tailwind CSS 还原这个页面。
注意：
1. 使用语义化 HTML
2. 响应式设计（移动端优先）
3. 动画效果用 CSS transition
4. 图片用 Next.js Image 组件
```

Claude Code 生成的页面和设计稿的还原度大概在 85%。剩下 15% 需要手动微调间距、颜色等细节。但这已经省了大量时间。

第三步，功能开发。产品列表的筛选、分页功能，联系表单的验证和提交，这些通用功能 Claude Code 写得很好，基本不用改。

第四步，CMS 集成。客户需要自己更新产品信息，我用了 Strapi 做 headless CMS。让 Claude Code 写了 Strapi 的 content type 定义和 Next.js 的数据获取逻辑。

**客户反馈**：满意。项目提前 3 天交付，客户给了 5 星评价。

### 案例 2：数据处理自动化

**需求**：一个电商客户每天要从 5 个供应商的 Excel 文件中汇总数据，生成统一格式的报表，然后上传到他们的 ERP 系统。之前都是人工做的，每天要花 2 小时。

**用时**：8 小时

**开发过程**：

```
帮我写一个 Python 脚本，功能：
1. 从指定文件夹读取 5 个不同格式的 Excel 文件
2. 解析每个文件的数据（我会提供每个文件的格式说明）
3. 数据清洗：去重、处理缺失值、统一格式
4. 合并成统一格式的汇总表
5. 生成 Excel 报表
6. 通过 API 上传到 ERP 系统

要求：
- 使用 pandas 处理数据
- 错误处理要完善，某个文件有问题不能影响其他文件
- 生成处理日志
- 支持定时运行（用系统 cron）
```

我把 5 个 Excel 的样例文件也发给了 Claude Code，它根据实际数据格式写了对应的解析逻辑。

这个项目报价 5,000 元。8 小时搞定，时薪 625 元。而且客户因为每天节省了 2 小时人工，非常满意。

### 案例 3：微信小程序

**需求**：一个本地餐饮店要一个简单的点餐小程序。

**用时**：35 小时（这是个稍大的项目）

**用 AI 的地方**：

- 小程序页面组件生成（节省约 50% 的前端开发时间）
- 后端 API 开发（Node.js + MySQL）
- 微信支付集成的代码骨架
- 订单状态机的逻辑

**注意**：微信支付和小程序的一些特殊 API，Claude Code 的知识有时候不够新。这些地方我需要参考微信官方文档手动调整。

这个项目报价 15,000 元，35 小时交付。

## 用 AI 接私活的工作流

### 需求分析阶段

客户的需求往往是模糊的。我会先跟客户详细沟通，然后用 Claude Code 帮我整理：

```
以下是客户的需求描述（原文）：
[粘贴客户的消息]

请帮我整理成结构化的需求文档，包括：
1. 功能需求列表（按优先级排序）
2. 非功能需求（性能、安全等）
3. 需要客户确认的问题清单
4. 技术方案建议
```

这样既帮我理清了思路，也能发给客户确认，避免后期扯皮。

### 开发阶段

我的典型开发流程：

1. **让 Claude Code 搭骨架**：项目初始化、路由、数据库 schema、基础组件
2. **让 Claude Code 写功能模块**：每次给一个明确的功能描述，让它生成代码
3. **我来审查和调试**：检查代码逻辑、测试边界情况、修复 bug
4. **让 Claude Code 写测试**：单元测试和集成测试
5. **我来做最终测试和部署**

关键原则：**不要完全依赖 AI 生成的代码，一定要自己审查**。我遇到过几次 AI 生成的代码有安全漏洞（比如 SQL 注入、XSS），好在我在 code review 时发现了。

### 沟通阶段

客户沟通也能用 AI 辅助：

- 需求确认邮件的起草
- 技术方案文档的撰写
- 项目进度报告的整理
- 英文客户的邮件翻译和润色

### 交付阶段

交付前我会让 Claude Code 帮我：

- 生成部署文档
- 写 README
- 做代码注释和清理
- 生成简单的用户使用说明

## 接私活要注意的坑

### 1. 需求蔓延（Scope Creep）

这是接私活最大的坑。客户说"就加个小功能"，结果这个"小功能"要花 10 个小时。

防范方法：
- 项目开始前明确写清楚需求范围
- 超出范围的需求单独报价
- 合同里写清楚"X 次免费修改，超出按小时计费"

### 2. 尾款收不回来

我吃过一次亏。做完了客户说"先上线测试几天，没问题就付尾款"，然后就没然后了。

现在我的付款方式是：
- 30% 预付款，开工
- 40% 中期交付测试版时
- 30% 最终交付时

Upwork 这类平台有 escrow（资金托管）机制，可以避免这个问题。

### 3. 时间估算不准

用 AI 之后，估算更难了。因为有些任务 AI 能秒解，有些任务 AI 帮不上忙（比如复杂的业务逻辑、第三方 API 的集成调试）。

我的做法是估算完之后加 30% 的缓冲。宁可提前交付给客户惊喜，也不要延期让客户失望。

### 4. 不要同时接太多活

我试过同时接 4 个项目，结果每个都延期了。现在我同时最多做 2 个项目，确保每个都能高质量交付。

### 5. AI 生成代码的版权问题

这是一个灰色地带。目前法律上还没有明确的规定，但我的原则是：
- 不直接告诉客户我用了 AI（就像你不会告诉客户你用了什么 IDE）
- 确保最终交付的代码经过我的审查和修改
- 不要把带有明显 AI 痕迹的代码直接交付

### 6. 保持技术能力

用 AI 久了容易变得依赖。但如果你不理解 AI 生成的代码，你就无法调试问题、做 code review、跟客户解释技术方案。

我的做法是每周花几个小时学习新技术，保持自己的基本功。AI 是工具，不是替代。

## 给想开始接私活的朋友的建议

### 1. 先从小项目开始

不要一上来就接大项目。先从 1,000-3,000 元的小活做起，积累经验和口碑。

### 2. 建立作品集

做几个个人项目放到 GitHub 和个人网站上。客户会看你之前做过什么。

### 3. 选择一个细分方向

"什么都能做"等于"什么都不精"。选一个方向深耕，比如"React 企业官网开发"或"Python 数据处理自动化"。

### 4. 学会说不

不是所有活都值得接。预算太低、需求不清、客户沟通困难的活，果断拒绝。时间花在高价值的项目上。

### 5. 持续学习

技术在变，AI 工具在进化，市场需求在变。保持学习，才能保持竞争力。

## 总结

用 AI 接私活，核心逻辑是：AI 帮你压缩了开发时间，但你按市场价格报价，中间的差价就是你的效率溢价。

但这不意味着你可以偷懒。AI 生成的代码需要你审查、调试、优化。客户的需求需要你理解、沟通、管理。项目的质量需要你把关。

AI 是一个强大的加速器，但驾驶这辆车的人还是你自己。
