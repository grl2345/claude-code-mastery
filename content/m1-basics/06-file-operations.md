## Claude Code 是怎么「看到」你的项目的？

**Claude Code 不会一启动就读完整个项目，它采用三步惰性读取机制：**

1. **目录扫描**：读取文件树，了解项目整体布局，但不读文件内容——相当于"看了一眼文件夹结构"
2. **按需读取**：当你的需求涉及某个文件时，它才会打开来看
3. **上下文积累**：读过的文件留在对话的上下文中，后续对话可以直接引用

> 好处是节省上下文窗口的空间；代价是你不主动指出相关文件时，Claude Code 可能会遗漏重要信息。很多时候结果不理想，不是 Claude Code 能力不行，而是它根本没看到关键的那个文件。

## 如何利用现有代码作为「模板」快速生成新模块？

**直接指向一个现有模块说"照这个来"，比手动描述代码风格高效得多。** 这是 Claude Code 最实用的技巧之一：

```bash
$ "参考 src/modules/posts/ 的代码结构和风格，
   为 comments 创建同样结构的新模块，
   包含 routes.ts、service.ts、types.ts"
```

Claude Code 会自动从参考代码中学习命名规范、错误处理方式、文件组织结构等细节。项目里只要有一个模块的代码是你满意的，后续所有类似模块都可以用这个方式快速生成。**前期在第一个模块上多花点时间打磨好，后面就是复利。**

## 修改多个文件时如何避免遗漏？

**提前告诉 Claude Code 哪些文件需要一起改。** 不指定的话，它可能只改了类型定义，忘了更新 service 层和测试：

```bash
$ "修改 User 接口，添加 avatar 字段。
   请同时更新：
   - src/modules/user/service.ts
   - src/modules/user/routes.ts
   - tests/modules/user/service.test.ts"
```

Claude Code 不是故意遗漏，只是没有意识到这些文件之间的关联。明确指定关联文件，就能避免大多数遗漏问题。

> 养成审查每次修改的习惯。就像你 review 同事代码一样——这不是不信任工具，而是对自己的代码负责。
