# Claude Code 要花多少钱？计费模式详解与 7 个省钱技巧

**Claude Code 有两种计费模式：API 按量付费（按 Token 计费）和 Max 订阅（$20-$200/月固定费用）。** 掌握省钱技巧，能在不影响效率的前提下降低 35-40% 的开支。

---

## Claude Code 有哪两种计费模式？

### API 按量付费

通过 Anthropic API 调用 Claude 模型，按 Token 数量计费：

```
费用 = 输入 Token 数 × 输入单价 + 输出 Token 数 × 输出单价
```

**Claude Sonnet 定价：**

| 项目 | 单价 |
|------|------|
| 输入 Token | $3 / 百万 Token |
| 输出 Token | $15 / 百万 Token |

**Claude Opus 定价：**

| 项目 | 单价 |
|------|------|
| 输入 Token | $15 / 百万 Token |
| 输出 Token | $75 / 百万 Token |

### Max 订阅计划

| 计划 | 月费 | 说明 |
|------|------|------|
| Pro | $20/月 | 包含一定量的 Claude Code 使用 |
| Max 5x | $100/月 | Pro 的 5 倍用量 |
| Max 20x | $200/月 | Pro 的 20 倍用量 |

## 一次 Claude Code 对话大概要花多少钱？

**Token 是模型处理文本的最小单位。** 粗略估算：英文 1 词约 1-2 Token，中文 1 字约 1.5-2 Token，1 行代码约 5-15 Token。

| 场景 | 输入 Token | 输出 Token | 估算费用 (Sonnet) |
|------|-----------|-----------|-----------------|
| 简单问答 | 1,000 | 500 | $0.01 |
| 读取文件并修改 | 10,000 | 2,000 | $0.06 |
| 大型重构 | 50,000 | 10,000 | $0.30 |
| 全项目分析 | 200,000 | 20,000 | $0.90 |

## 为什么对话越长费用越高？

**Claude Code 每次交互会自动携带全部上下文信息，** 包括对话历史、读取过的文件、项目结构和 CLAUDE.md 内容。这意味着对话越长，每次请求的输入 Token 越多，**一个持续 30 轮的对话，后期每轮的输入 Token 可能是前期的 10 倍以上。**

---

## API 按量付费和订阅该怎么选？

**根据月度 API 费用判断：低于 $20 选按量付费，$20-$100 选 Pro/Max 5x，超过 $100 选 Max 订阅。**

| 适合 API 按量付费 | 适合 Max 订阅 |
|-----------------|-------------|
| 使用频率不固定，有时一周不用 | 每天都在用 Claude Code |
| 需要精确控制成本 | 不想操心费用计算 |
| 团队使用，需要分账 | 个人开发者，追求简单 |
| 只用于特定项目 | 用量较大且稳定 |

**盈亏平衡点：**

- 月度 API 费用 < $20 → 按量付费更经济
- 月度 API 费用 $20-$50 → Pro 计划更合适
- 月度 API 费用 > $100 → Max 5x 更划算

> 建议：先用 API 按量付费一个月，统计实际用量，再决定是否切换到订阅。

## 有哪些实用的省钱技巧？

以下 7 个技巧经过实践验证，能帮你降低 35-40% 的 Token 开支：

### 技巧一：精简提示词，像写电报一样

```
# 差：约 80 Token
你好, 我想请你帮我看一下这个文件, 主要功能是处理用户登录,
但是我发现有一个 bug, 就是当用户输入空密码的时候会崩溃...

# 好：约 20 Token
修复 src/auth/login.ts 中空密码导致崩溃的 bug
```

**核心原则：提示词要像电报，信息密度高，废话少。** 单次节省不多，但累计差距明显。

### 技巧二：每 10-15 轮用 /compact 压缩上下文

```bash
/compact
```

**上下文膨胀是费用增长的主要原因。** `/compact` 让 Claude 总结之前的对话，用更少的 Token 表达相同的信息。建议在以下时机使用：

- 每 10-15 轮对话执行一次
- 切换任务主题时
- 感觉响应变慢时（通常意味着上下文过长）

### 技巧三：拆成多个短对话，而非一个长对话

**每个短对话的上下文都从较小的基数开始，总体 Token 消耗更少。**

```
对话 1：分析现有代码结构，输出重构方案
对话 2：按方案重构核心逻辑
对话 3：编写测试
对话 4：更新文档
```

### 技巧四：用 CLAUDE.md 替代重复说明

**CLAUDE.md 会被自动加载且只加载一次，比每次手动输入省得多。**

```markdown
# 项目说明
- 技术栈: Next.js 14 + TypeScript + Prisma
- 代码规范: 使用 ESLint airbnb 配置
- 测试框架: Vitest
- 命名: 组件用 PascalCase, 函数用 camelCase
```

### 技巧五：80% 任务用 Sonnet，难题再切 Opus

| 任务类型 | 推荐模型 | 原因 |
|---------|---------|------|
| 简单代码生成 | Sonnet | 速度快，费用低 |
| 复杂架构设计 | Opus | 推理能力强 |
| 代码解释 | Sonnet | 足够胜任 |
| Bug 调试 | Sonnet/Opus | 视复杂度而定 |
| 文档撰写 | Sonnet | 性价比高 |

```bash
/model sonnet
/model opus
```

### 技巧六：精确指定文件路径和行号

```
# 差：Claude Code 会扫描大量不相关文件
修复登录页面的样式问题

# 好：精确定位，减少文件读取
修复 src/pages/Login.tsx 第 45-60 行的 flexbox 布局问题
```

### 技巧七：利用 Prompt Caching 机制

**Anthropic API 的缓存读取比标准输入价格低 90%。** Claude Code 会自动利用缓存，你需要做的是：

- 保持 CLAUDE.md 内容稳定，不要频繁修改
- 在同一个会话中处理相关任务（缓存在会话内有效）
- 避免在提示词开头加入变化的内容（缓存按前缀匹配）

## 如何监控 Claude Code 的费用？

### 查看当前会话用量

```bash
/cost
```

### 在 API 控制台设置费用告警

登录 Anthropic Console（Settings → Usage Limits → Set monthly budget alert），可以查看每日/每月调用量、Token 消耗明细和费用趋势图。

### 设定预算上限

API 模式下可以设置**硬性消费上限**：月度上限（达到后 API 自动停止响应）和每日上限（防止某天异常消耗）。

## 团队使用 Claude Code 如何控制成本？

**三个核心策略：共享配置、沉淀知识、分级使用。**

1. **共享 CLAUDE.md**：一份好的 CLAUDE.md 让每个成员减少重复提示，整体省下大量 Token
2. **知识沉淀**：把 Claude Code 生成的好方案记录下来，形成团队知识库，避免重复消耗 Token
3. **使用场景分级**：

| 级别 | 场景 | 策略 |
|------|------|------|
| 必要 | 复杂 Bug 调试 | 放心使用 |
| 推荐 | 代码审查辅助 | 控制频率 |
| 可选 | 简单代码生成 | 能手写就手写 |
| 避免 | 闲聊、测试玩耍 | 使用免费工具 |

## 使用 Claude Code 有哪些常见的费用陷阱？

**四个最容易踩的坑：**

1. **无限循环对话**：Claude Code 自动修 Bug 时可能陷入"修了又错、错了又修"的循环——设置合理的迭代上限
2. **大文件读取**：读取 5000 行文件一次消耗大量 Token——指定行范围比读取整个文件高效得多
3. **频繁切换任务**：同一对话中切换不相关任务，上下文充满无关信息——不同任务开不同对话
4. **忘记关闭会话**：长时间不用的会话再次使用时带着很长的历史上下文——及时 `/compact` 或开新对话

## 省钱的核心原则是什么？

**一句话：减少不必要的 Token 消耗。** 行动清单：

1. 确认自己适合 API 还是订阅模式
2. 写好 CLAUDE.md，避免重复说明
3. 养成写精简提示词的习惯
4. 每 10-15 轮用 `/compact` 压缩上下文
5. 日常任务用 Sonnet，难题切 Opus
6. 指定文件路径，避免全局扫描
7. 每周检查一次费用报告，发现异常及时调整

掌握这些技巧后，大多数开发者可以将月度费用控制在合理范围内，同时不影响开发效率。
