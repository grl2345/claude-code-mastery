## AI 的建议什么时候该听、什么时候不该听？

**用「影响范围 x 回退难度」矩阵判断：影响越大、越难回退，越不能交给 AI。**

## 决策分类矩阵

|  | 影响小（局部） | 影响大（全局） |
|--|---------------|---------------|
| **容易回退** | 放心交给 AI：变量命名、小函数抽取 | 可以参考 AI：模块内重构 |
| **难以回退** | 自己判断为主：第三方库选型 | 必须自己决策：数据库、架构模式 |

**影响越大、越难回退，越不能交给 AI。**

## 哪些决策适合交给 AI？

**代码级优化。** 局部改写，容易回退。

**模式识别。** 但要判断模式是否适合你的场景。

**技术对比。** AI 列出优劣，你做最终选择。

## 哪些决策必须自己拿主意？

**技术栈选型。** 团队能力、预算、维护计划 AI 不了解。

**架构模式。** 需要综合考虑业务增长、团队规模、运维能力。

**取舍决策。** 性能 vs 可读性？没有标准答案。

> **AI 是参谋，不是将军：** 让 Claude Code 给你信息、分析利弊、列出选项。但做决定的人是你。

## 怎样让 AI 辅助决策而不替你做决策？

**让 Claude Code 列出选项和优缺点对比，但在 prompt 里明确说「不要替我做选择」。**

```bash
$ "对比 Zustand、Jotai、Redux Toolkit：
   - 学习曲线 / 适合规模 / TS 支持
   列出优缺点，不要替我做选择"
```
