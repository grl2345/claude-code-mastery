## 怎样把方法论从理论变成肌肉记忆？

**用一份覆盖 5 个阶段的速查检查清单，反复执行直到变成自然反应。** 下面是我日常在用的清单：

```
═══ 开始前 ═══
□ CLAUDE.md 是否更新？
□ 需求是否用三层模型拆解过？
□ 目标 → 步骤 → 验证 是否都明确？

═══ 执行中 ═══
□ 一次对话只做一件事
□ 一次 prompt 只做一个步骤
□ 精确引用文件
□ 每步产出后审查（2 分钟内能审完）
□ 审查通过后立即 commit

═══ 审查时 ═══
□ 导入和依赖存在吗？
□ 边界情况处理了吗？
□ 错误处理合理吗？
□ 有没有过度设计？

═══ 遇到问题 ═══
□ 小瑕疵 → 直接修
□ 逻辑偏差 → 指出问题，重写
□ 方向错误 → git reset，新对话

═══ 结束时 ═══
□ 测试覆盖了关键路径吗？
□ commit message 规范吗？
□ 有经验可以加到 CLAUDE.md？
```

## 怎样建立持续改进的正反馈循环？

**通过「记录 → 沉淀 → 分享」三步循环，让每次经验都转化为长期资产。**

**步骤一：记录。** 每次犯的错误、踩的坑、发现的最佳实践。

**步骤二：沉淀。** 定期回顾，把高频模式写进 CLAUDE.md。

**步骤三：分享。** 教别人会加深自己的理解。

## 怎样把个人经验推广到团队？

**通过 3 个抓手统一团队的 AI 协作标准。**

**抓手一：统一 CLAUDE.md。** 团队共享，确保规范一致。

**抓手二：共享 prompt 库。** 好用的 prompt 模式收集起来。

**抓手三：AI 代码审查标准。** 写进 code review 指南。

## 思维篇 12 课的核心要点是什么？

**任务拆解（1-3 课）**：三层模型 + 分治策略。核心是想清楚再动手。

**上下文工程（4-6 课）**：上下文是稀缺资源，五个策略管理。CLAUDE.md 一次投入长期受益。

**渐进式开发（7-9 课）**：小步快跑，每步审查，保持决策权。

**质量保障（10-12 课）**：测试不再可选。架构决策不能外包。方法论变成习惯。
